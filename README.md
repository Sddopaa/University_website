# Веб-сайт Политехнического университета

Веб-приложение для управления учебным процессом в университете на основе Spring Boot.

## Описание проекта

Приложение предоставляет функциональность для трех типов пользователей:
- **Студенты** - просмотр заданий, профиль
- **Преподаватели** - создание заданий, управление студентами
- **Администраторы** - управление пользователей, изменение ролей

## Технологии

- **Backend:** Spring Boot 3.x, Spring MVC, Spring Data JPA, Spring Security
- **Frontend:** Thymeleaf, HTML, CSS, JavaScript
- **База данных:** PostgreSQL
- **Сборка:** Maven
- **Java:** 17+
- **Сервер:** Встроенный Tomcat


**Ключевые файлы и их назначение:**

1. **Контроллеры (`controllers/`)**:
    - `AdminController.java` - обработка запросов администратора
    - `AuthController.java` - аутентификация и регистрация
    - `MainController.java` - основные страницы сайта
    - `ProfileController.java` - управление профилем
    - `StudentController.java` - функционал студента
    - `TeacherController.java` - функционал преподавателя

2. **Модели (`models/`)**:
    - `User.java` - базовая модель пользователя
    - `Student.java` - модель студента (наследует User)
    - `Teacher.java` - модель преподавателя (наследует User)
    - `Admin.java` - модель администратора (наследует User)
    - `Task.java` - модель задания

3. **Репозитории (`repository/`)**:
    - `UserRepository.java` - доступ к данным пользователей
    - `StudentRepository.java` - доступ к данным студентов
    - `TeacherRepository.java` - доступ к данным преподавателей
    - `AdminRepository.java` - доступ к данным администраторов
    - `TaskRepository.java` - доступ к данным заданий

4. **Сервисы (`services/`)**:
    - `UserService.java` - бизнес-логика пользователей
    - `TaskService.java` - бизнес-логика заданий

5. **Конфигурация (`config/`)**:
    - `SecurityConfig.java` - настройки безопасности Spring Security

6. **Ресурсы (`resources/`)**:
    - `templates/` - HTML шаблоны Thymeleaf
    - `static/` - статические файлы (CSS, JS, изображения)
    - `application.properties` - конфигурация приложения

## Функциональность

### Аутентификация и авторизация
- Регистрация пользователей с разными ролями (студент, преподаватель, администратор)
- Вход в систему с проверкой логина и пароля
- Разграничение прав доступа на основе ролей
- Сессионное управление авторизацией

### Профиль пользователя
- Загрузка и обновление аватара
- Установка аватара по умолчанию
- Хранение изображений в директории `static/images/userAvatar/`
- Просмотр личных данных

### Управление заданиями (преподаватели)
- Создание заданий для студентов своего факультета
- Валидация данных заданий (заголовок 3-50 символов, описание 20-200 символов)
- Удаление собственных заданий
- Просмотр списка созданных заданий
- Выбор студентов по факультету

### Просмотр заданий (студенты)
- Просмотр заданий, назначенных преподавателями
- Отображение предмета, преподавателя и описания задания
- Сортировка и фильтрация заданий

### Административная панель
- Изменение ролей пользователей (студент ↔ преподаватель)
